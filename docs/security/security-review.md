# Security-review

Purpose

This document reviews implemented security controls and links controls to risks and scan results. Content aligns with automated Checkov validation and AWS security guidance.

Validation approach

Terraform code scanned using Checkov.
Controls verified across encryption, logging, network protection, and availability.
Scan evidence stored under scans/checkov.

Control domains reviewed

Transport security.
Web application firewall protection.
Logging and monitoring.
Data protection.
DNS security.
Resilience and failover.
